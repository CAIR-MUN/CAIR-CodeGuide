Security is essential in software development, especially in data center
environments where sensitive information and critical operations are
involved. This chapter summarizes developer-focused security guidance.

## 9.1 Input Validation and Sanitization

- Treat all inputs as untrusted. Validate type, length, format, and
  allowed values.
- Prefer whitelisting over blacklisting.

### Example (Python - simple email validation)

```python
import re

def is_valid_email(email: str) -> bool:
    return bool(re.match(r"^[\w.-]+@[\w.-]+\.[a-zA-Z]{2,}$", email))
```

## 9.2 Prevent Injection

- Use parameterized queries / prepared statements for DB access.
- Escape or encode outputs for web contexts (HTML, JavaScript) to
  prevent XSS.

### Example (Python + sqlite3)

```python
import sqlite3
conn = sqlite3.connect('users.db')
cur = conn.cursor()
cur.execute('SELECT * FROM users WHERE id = ?', (user_id,))
```

## 9.3 Authentication & Secrets

- Hash passwords with Argon2 or bcrypt; never store plain-text passwords.
- Use secure secret storage (vaults or environment-protected files).

## 9.4 Data Encryption

- Encrypt in transit (TLS) and at rest where required.
- Avoid logging secrets; mask or redact sensitive fields.

## 9.5 OWASP Top 10 (Developer Summary)

- Injection, Broken Authentication, Sensitive Data Exposure, XXE,
  Broken Access Control, Security Misconfiguration, XSS, Insecure
  Deserialization, Vulnerable Components, Insufficient Logging.

## 9.6 Practical Checklist

- Run static analysis and dependency checks.
- Use automated security tests in CI.
- Rotate secrets and patch dependencies promptly.
